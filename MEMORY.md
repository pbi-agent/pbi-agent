- Provider/runtime rules: outbound provider and tool HTTP must use `urllib.request`; provider abstractions live under `src/pbi_agent/providers/`; internal data belongs in `~/.pbi-agent/`; no migration or backward-compatibility shims should be added.
- Current product/provider shape: `chatgpt` and `github_copilot` are first-class subscription-backed provider kinds, distinct from API-key `openai`; provider-auth UI is metadata-driven; ChatGPT/Copilot account flows are implemented across CLI and web settings.
- Model discovery: settings-side model selection is runtime provider discovery, not the local catalog. Discovery supports ChatGPT Codex backend, Google Gemini, generic OpenAI-compatible `/models`, and GitHub Copilot `GET /models`; Copilot’s live default model is `gpt-5.4`.
- Project skills: `pbi-agent skills list` and `pbi-agent skills add` are top-level project-skill commands. Installer logic lives in `src/pbi_agent/skills/project_catalog.py` and `project_installer.py`, supports public GitHub shorthand/repo/tree sources plus local filesystem sources, uses authenticated GitHub archive download with git fallback when needed, only treats root `SKILL.md` and public `skills/*/SKILL.md` entries as installable bundles, and user-facing lists hide manifest paths.
- Project commands: `pbi-agent commands list` and `pbi-agent commands add` manage `.agents/commands/*.md` prompt presets. Command imports use the shared GitHub/local source materializer in `src/pbi_agent/project_sources.py`, default to public `commands/*.md` discovery, ignore repo-root `.agents/commands/*` unless explicitly targeted, and install normalized files into `.agents/commands/<slug>.md`. The official remote catalog is `pbi-agent/commands`.
- Project agents: `pbi-agent agents list` and `pbi-agent agents add` manage `.agents/agents/*.md` sub-agent definitions. Agent imports reuse the shared GitHub/local source materializer, default to public `agents/*.md` discovery, ignore repo-root `.agents/agents/*` unless explicitly targeted, and install normalized files into `.agents/agents/<name>.md`. The official remote catalog is `pbi-agent/agents`.
- Project sub-agents: discover only `.agents/agents/*.md`. Supported frontmatter is limited to `name` and `description`; sub-agent files are model-agnostic and child runtime selection comes from the active profile/CLI defaults plus `sub_agent_model`.
- Frontmatter: shared parser lives in `src/pbi_agent/frontmatter.py`, supports folded/literal YAML block scalars for `description` only, strips actual literal block indent, and rejects block-scalar `name`. Project agent install rejects empty, dot-segment, absolute, backslash, nested, or traversal names before writing `.agents/agents/<name>.md`.
- Project commands: `.agents/commands/plan.md` is non-interactive, forbids clarifying questions and `request_user_input`, and emits plain Markdown plans. `.agents/commands/review.md` emits plain Markdown review output, not JSON. `.agents/commands/execute.md` is compact prose. The local `.agents/skills/compress` skill is prompt-only: no scripts, no network.
- Branding: keep technical identifiers such as `pbi-agent`, `pbi_agent`, `PBI_AGENT_*`, and `~/.pbi-agent`, but position public product copy as a general local coding agent. Use canonical packaged logo asset `src/pbi_agent/web/static/logo.jpg` for web/docs/favicon routes, with `/favicon.ico`, `/favicon.png`, and `/logo.png` compatibility fallbacks. Startup banner uses the dark `BI` monogram, `work smart.`, centered rows, and centered `v{__version__}`.
- Web/session behavior: saved web sessions reopen with `resume_session_id`; `_resume_session()` restores saved provider checkpoints when transcript replay is unavailable. `WebSessionManager` startup uses a `SessionStore` workspace lease with heartbeat; lease acquisition is atomic with SQLite `BEGIN IMMEDIATE` plus `busy_timeout`, and running-task normalization happens during managed app startup to avoid false failed kanban tasks.
- Provider request ordering: standard OpenAI Responses sessions keep system prompt in `instructions` and rely on `previous_response_id` for multi-turn continuity. ChatGPT subscription backend prepends system prompt to replay/current input so replay starts system-first without duplicating standard Responses behavior. Verified against `~/codex` and `~/opencode`: for ChatGPT HTTP/SSE, `x-codex-turn-state` is sticky routing state, not an incremental-input contract; do not send tool-result-only deltas on that header alone.
